# Анализ рисков ОСФ: Матрица Риск × Рейтинг РУСАДА

## Контекст и аудитория

Дополнительный аналитический слой поверх основной модели рисков (ipynb). Целевая аудитория — работники антидопинговой сферы. Объект анализа — **виды спорта** через призму ОСФ, **не регионы**. Выдача — вход для управленческих решений по матрице приоритетов.

## Критерии рейтинга (9 штук)

`Стратегия` · `План-график` · `Регионы` · `Сайт` · `Семинар` · `Соглашение` · `Допуск` · `Мониторинг` · `Инфо`

Итоговый балл 0–100. Критерий считается **выполненным при значении ≥ 1** (в PDF встречается значение `2`).

## Зоны риска (из ipynb)

В логике и данных — только текстовые коды. Эмодзи допустимы исключительно как декор в Excel и дашборде.

```
RED    — lag_1q + lag_2q > 0              (свежая активность)
ORANGE — rolling_mean_8q >= 0.25
         ИЛИ rolling_sum_4q >= 2          (историческая устойчивость)
GREEN  — всё остальное
```

Порядок проверки: RED → ORANGE → GREEN. Строка получает **ровно одну** зону.

Соответствие терминов матрицы:
- «Систематичность нарушений» = RED или ORANGE
- «Нет систематичности» = GREEN

## Матрица приоритетов (порог рейтинга = 80 баллов)

| Приоритет | Квадрант | Рекомендация |
|---|---|---|
| **1** | Рейтинг < 80 + RED/ORANGE | Приоритизировать работу по невыполненным критериям рейтинга |
| **2** | Рейтинг ≥ 80 + RED/ORANGE | Усилить антидопинговое образование; беседы с заинтересованными сторонами; рассмотреть внесоревновательное тестирование |
| **3** | Рейтинг < 80 + GREEN | Продолжать профилактику; приоритизировать работу по невыполненным критериям рейтинга; превентивная работа с федерацией и регионом |
| **4** | Рейтинг ≥ 80 + GREEN | Поддерживать текущий уровень; мониторинг; плановые проверки |

### Шаг 3 — Матчинг источников

**Нормализация** (применяется к обеим сторонам перед сравнением):
1. Нижний регистр.
2. Удалить слова: «общероссийская», «всероссийская», «федерация», «союз», «ассоциация», «России», «российская», кавычки, лишние пробелы.
3. Точное совпадение нормализованных строк.

**ALIASES-словарь** (для расхождений формулировок, применяется до нормализации):
- «плавание» → «водных видов спорта»
- «футбол» → «футбольный союз»
- «биатлон» → «биатлонистов»

**Запрещено** наивное вхождение подстроки — даёт ложные соответствия («бокс» ⊂ «кикбоксинг», «борьба» ⊂ «борьба на поясах»). Алгоритм: ALIASES → полное покрытие токенов → ослабленный частичный токен (только как резерв, с пометкой низкой уверенности).

Несопоставленные строки — в лист **«Не сопоставлено»** (название, источник, причина). Молчаливое удаление запрещено.

**Инвариант полноты:** строки итоговой таблицы + строки «Не сопоставлено» (сторона рейтинга) = N_рейтинг. Вывести это равенство с числами.

### Шаг 4 — Квадрант и приоритет

Присвоить каждому виду спорта приоритет 1–4 по матрице выше.

**Краевые случаи:**
- proba = NaN → зона по lag/rolling-условиям; в «Оценка риска (proba)» писать «нет данных», строку не удалять.
- Вид спорта есть в рейтинге, нет в модели → квадрант по рейтингу, зона «нет данных модели», приоритет как для GREEN (3 или 4), обоснование: «данных модели нет».
- Вид спорта есть в модели, нет в рейтинге → в «Не сопоставлено», в основную таблицу не включать.
- 0 выполненных или 0 невыполненных критериев → писать «нет», ячейку пустой не оставлять.

### Шаг 5 — Рекомендации и обоснования

Рекомендация — дословно из таблицы приоритетов Шага 4.

Обоснование:
- RED/ORANGE: «Рисковый из-за [какие lag/rolling-условия сработали и какими значениями]; нехватка баллов по критериям: [перечислить]» (если все выполнены — «критерии рейтинга выполнены полностью»).
- GREEN: «Низкий риск по модели (proba: [значение]); [состояние рейтинга: набранные баллы / недостающие критерии]».

### Шаг 6 — Самопроверка (обязательна перед выдачей)

Вывести каждый инвариант с фактическими числами:

1. Строки итоговой таблицы + «Не сопоставлено» (сторона рейтинга) = N_рейтинг.
2. Сумма строк по четырём приоритетам = число строк итоговой таблицы.
3. Каждая строка имеет ровно одну зону из {RED, ORANGE, GREEN, «нет данных модели»} и ровно один приоритет 1–4.
4. Нет строк RED/ORANGE с приоритетами 3–4; нет строк GREEN с приоритетами 1–2.
5. Нет строк с рейтингом ≥ 80 в приоритетах 1 и 3; нет строк с рейтингом < 80 в приоритетах 2 и 4.
6. Распределение строк по зонам (RED/ORANGE/GREEN — по сколько). Если зона содержит 0 строк — взять 3 случайные строки, вручную посчитать lag/rolling и показать, почему условие не сработало.
7. Нет региональных названий в колонке «Вид спорта (ОСФ)».
8. Все строки содержат официальное название ОСФ (не сокращение).
9. Итоговая таблица отсортирована: приоритет по возрастанию → RED раньше ORANGE → proba по убыванию.
10. На дашборде точки не перекрываются (jitter применён), подписи читаемы.

Если хотя бы один инвариант нарушен — исправить и повторить самопроверку. Выдавать результат с нарушенным инвариантом запрещено.

Все сопоставления фиксируются на листе **«Аудит матчинга»** (исходное имя, ОСФ, тип матчинга, уверенность, зона).
