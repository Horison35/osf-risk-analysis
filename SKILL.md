# Анализ рисков ОСФ: Матрица Риск × Рейтинг РУСАДА

## Контекст и аудитория

Дополнительный аналитический слой поверх основной модели рисков (ipynb). Целевая аудитория — работники антидопинговой сферы. Объект анализа — **виды спорта** через призму ОСФ, **не регионы**.

| PDF «Рейтинг ОСФ 2025» | **142 федерации**, 9 критериев, баллы 0–100 | https://rusada.ru/federations_leagues/raiting/ → «Рейтинг ОСФ 2025» |
| ipynb модели рисков | Прогнозы нарушений по виду спорта × регион | Передаётся пользователем |

## Критерии рейтинга (9 штук)

`Стратегия` · `План-график` · `Регионы` · `Сайт` · `Семинар` · `Соглашение` · `Допуск` · `Мониторинг` · `Инфо`

Итоговый балл 0–100. **Важно:** значения критериев НЕ строго бинарны — в PDF встречается значение `2` (например, «План-график» у ряда федераций). Поэтому критерий считается **выполненным при значении ≥ 1**, а не только при `== 1`.

## Логика агрегации рисков из ipynb

Строго по коду модели (`отчет_виды_и_топ_регион`):

```python
# Зона для каждой строки (вид спорта × регион):

# 🟠 ОРАНЖЕВАЯ — rolling_mean_8q >= 0.25

# → берём наихудший регион как представителя вида спорта.
# Регион в итоговой таблице НЕ фигурирует.
```

## Полнота: в матрице ДОЛЖНЫ быть ВСЕ федерации

Итоговая таблица строится по **всему списку ОСФ из рейтинга** (142 строки), а не только по видам спорта, найденным в модели. Федерации, для которых в прогнозной модели нет данных, попадают в таблицу с зоной `⚪ НЕТ ДАННЫХ` и квадрантом `… + Риск неизвестен`. Их НЕЛЬЗЯ молча выбрасывать и НЕЛЬЗЯ считать рисковыми по умолчанию.

## Матчинг «вид спорта (модель) ↔ ОСФ (рейтинг)»

**НЕ использовать наивное вхождение подстроки** — это даёт ложные соответствия. Классический баг: `«бокс»` как подстрока входит в `«кикбоксинг»`, поэтому вид спорта «бокс» ошибочно распределяется вместе с кикбоксингом. Аналогично «борьба» пересекается с «борьбой на поясах».

Порядок матчинга в скрипте (`match_sport_to_osf`):

1. **Словарь алиасов** `ALIASES` (модель → фрагмент имени ОСФ) — для расхождений формулировок: «плавание» → «водных видов спорта», «футбол» → «футбольный союз», «биатлон» → «биатлонистов» и т. д. Значения прогоняются через `_norm()`, поэтому й/ё писать не критично.
2. **Полное покрытие значимых токенов** — токены вида спорта из модели должны быть **подмножеством** токенов ОСФ (гарантирует, что «бокс» ⊄ «кикбоксинг»).
3. **Ослабленный частичный токен** — только как крайний резерв, с пометкой низкой уверенности.

Все сопоставления фиксируются на листе **«Аудит матчинга»** (исходное имя, ОСФ, тип матчинга, уверенность, зона) — обязательно проверять его на спорных кейсах.

## Матрица квадрантов (порог рейтинга = 80 баллов)

| | Риск высокий (🔴/🟠) | Риск низкий (🟢) | Риск неизвестен (⚪) |
|---|---|---|---|
| **Рейтинг ≥ 80** | Усилить образование; беседы; внесоревновательное тестирование | Поддерживать, мониторить | Мониторинг; собрать данные для оценки |
| **Рейтинг < 80** | Приоритизировать невыполненные критерии рейтинга | Профилактика, подтянуть рейтинг | Подтянуть рейтинг; собрать данные |

## Алгоритм выполнения

### 1. Скачать PDF рейтинга
```bash
wget -O /tmp/Reiting-OSF.pdf \
  "https://rusada.ru/upload/iblock/75e/f67whlxj17072ibx26g1kew7rwbvmk1y/Rei_ting-OSF-2025-_itog_-_1_.pdf"
```
Если URL изменился — найти актуальную ссылку на странице rusada.ru/federations_leagues/raiting/.

### 2. Установить зависимость
```bash
sudo pip3 install pdfplumber -q
```
Скрипт сам извлекает таблицу из PDF через `pdfplumber` (не нужен промежуточный JSON). Строки таблицы приходят с 12 или 13 колонками (лишняя пустая ячейка), критерии всегда в позициях 1..9, СУММА — позиция 10, МЕСТО — позиция 11.

### 3. Запустить скрипт анализа
```bash
python /home/ubuntu/skills/osf-risk-analysis/scripts/build_report.py
```
Перед запуском обновить в начале скрипта: `PDF_PATH`, `IPYNB_PATH`, `OUT_DIR`.
Если `IPYNB_PATH` отсутствует — скрипт отработает, пометив все ОСФ как `⚪ НЕТ ДАННЫХ`.

### 4. Самопроверка перед отправкой
- [ ] В итоговой таблице **все 142 ОСФ** (или актуальное число строк из PDF)
- [ ] Нет региональных названий в колонке «Вид спорта (ОСФ)»
- [ ] Лист «Аудит матчинга»: спорные кейсы верны (бокс ≠ кикбоксинг, борьба ≠ борьба на поясах)
- [ ] Квадранты согласованы с баллами и зоной; ⚪ не попадает в «Высокий риск»
- [ ] Дашборд: 3 графика, точки не перекрываются, серые ⚪-точки отложены ниже нуля
